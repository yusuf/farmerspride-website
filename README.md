# Farmer's Pride by AgroLink — Website Repository

## What this repo is

A structured scaffold for building the Farmer's Pride website using Grav CMS, Tailwind CSS, and Docker. It contains the brief, design system, content, and Docker infrastructure. The Grav site itself is generated by Claude Code in a build session.

## What Claude Code builds

Claude Code generates `grav/user/` — the only directory in a Grav installation that contains custom work:

```
grav/user/              ← generated by Claude Code
├── config/             ← site, system, and theme config YAML files
├── pages/              ← 5 Grav page .md files with frontmatter
└── themes/
    └── farmerspride/   ← Tailwind config, CSS, Twig templates, images
```

**Do not commit Grav core files.** Grav core (system/, vendor/, index.php, .htaccess, cache/, logs/, tmp/, bin/, backup/) is downloaded by Docker at build time and runs inside the container. It never belongs in this repo.

## For Claude Code

1. Read `BRIEF.md` completely
2. Read `DESIGN.md`
3. Read all 5 files in `content/`
4. Read `assets/IMAGE_MANIFEST.md`
5. Build `grav/user/` as specified in the brief

## Local development

```bash
# After Claude Code has built grav/user/:
cp grav/.env.example grav/.env
# Edit grav/.env with admin credentials
cd grav
docker-compose up --build
# Site: http://localhost:8080
# Admin: http://localhost:8080/admin
```

## Compile CSS

```bash
cd grav/user/themes/farmerspride
tailwindcss -i src/styles.css -o css/theme.css --minify
# Watch mode:
tailwindcss -i src/styles.css -o css/theme.css --watch
```

Commit `css/theme.css` — Docker uses it directly at container build time.

## Deploy to Coolify

- Source: this GitHub repo, `main` branch
- Dockerfile: `grav/Dockerfile`
- Build context: `grav/`
- Port: `80`
- Environment variables: `GRAV_ADMIN_USER`, `GRAV_ADMIN_PASS`, `GRAV_ADMIN_EMAIL`
- Enable auto-deploy on push to `main`
